# Markdown Style Guide GitHub Action

Automate the formatting and consistency of your Markdown files directly in your GitHub workflows.

## Features:
- Enforce standardized heading levels.
- Consistent bolding and italics.
- Smart handling of lists and code blocks.
- Customizable rule sets to fit your project's style.
- Automatically pushes formatted changes back to your repository.

## How It Works:
1. Add the action to your workflow (`.github/workflows/main.yml`).
2. Configure your desired markdown style rules.
3. On pull requests or pushes, the action will format your markdown files.
4. Optimized for technical writers and open-source maintainers.
